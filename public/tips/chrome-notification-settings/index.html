Chrome 通知不是你想的那样

先搞清楚 Chrome 里有三种不同类型的"通知"，很多人把这三者搞混了：

网站通知：网站通过 Chrome 的 Notifications API 发送到你桌面的弹窗消息。比如 Gmail 的新邮件提醒、微博的私信通知。这是本文重点。

Chrome 提示消息：Chrome 浏览器自身发的通知，比如"Chrome 已更新"、“下载完成”。在通知设置页面最底部有个"Chrome 可以发送通知"的开关。

系统通知：Windows/macOS 系统层面弹出的通知，跟 Chrome 的通知权限是两套独立体系。

关掉 Chrome 的网站通知设置，Chrome 自身的提示消息和系统通知可能还在响。这也是很多人抱怨"明明关了通知还是有弹窗"的原因。下面我会分别讲怎么关这三类。

最快的清理方式：一键清空所有通知权限

如果你之前瞎点了太多"允许"，通知列表已经乱成一锅粥，最快的方式是直接重置：

  1. 地址栏输入 chrome://settings/content/notifications
  2. 页面加载后往下滚到底部
  3. 点击「移除所有站点通知权限」按钮

这个操作不会关掉通知功能本身，只是把所有站点的权限重置为"默认"（也就是下次访问会重新询问）。相当于给通知列表来了个大扫除。

[Chrome 移除所有站点通知权限](/images/tips/chrome-notification-settings/img-2.jpg)

说实话，90% 的用户做到这一步就够了。但如果你有几十个允许通知的网站，重置后你会发现某些网站确实需要通知（比如在线文档协作时的实时更新提醒），这时候就需要下面更精细的管理方案了。

精确管理：逐站设置通知权限

重置之后，你可以按三种模式管理每个站点的通知：

  • 允许：网站可以直接给你发桌面通知，不会弹询问框
  • 屏蔽：彻底禁止该网站发送通知，即使网站请求也会被静默拒绝
  • 默认（询问）：网站请求通知时 Chrome 会弹出询问框让你选

打开 chrome://settings/content/notifications，你会看到两个分区——上面是"允许"列表（已授权的网站），下面是"屏蔽"列表（已拒绝的网站）。每个网站右边有三个按钮：允许、屏蔽、移除（恢复到默认）。

建议的处理策略：

把通知网站分成三类来处理：

类别处理方式示例
必须留（工作/安全类）保持允许Gmail、日历、Notion、银行网站
可以留但不想被打扰保持允许，后续改系统音量微博、B站、Twitter
纯垃圾广告移到屏蔽各种新闻站、购物站、工具站

这里有个很多人不知道的技巧：Chrome 的通知设置支持搜索。在页面顶部的搜索框输入域名，可以快速定位到某个网站，不用在几十条列表里慢慢翻。如果你只想改某个站的通知权限，搜域名比翻列表快得多。

一刀切方案：彻底关闭网站通知

如果你根本不想用 Chrome 的通知功能（邮件用手机 App 看，社交用独立客户端），那直接关掉：

chrome://settings/content/notifications → 把顶部的「网站可以发送通知」开关关掉

关掉之后所有效果：

  • 已经允许的网站通知全部消失
  • 新网站请求通知会被自动拒绝，不再弹询问框
  • Chrome 提示消息（浏览器更新、下载完成）不受影响
  • 系统通知（Windows 通知中心）不受影响

这个方案适合两类人：一是从来不用网站通知的用户，二是帮家里老人/小孩设置电脑时减少干扰源。如果你日常使用 Gmail 网页版或有在线协作需求，不建议用这个方案。

顺便说一句，很多人以为关了 Chrome 通知就万事大吉了，结果电脑右下角还是有弹窗。那多半是系统通知或者 Chrome 自身的提示消息在响。这两个的关法不同，下面讲。

额外清理：Chrome 提示消息和系统通知

Chrome 自身提示消息

设置路径：chrome://settings/content/notifications → 页面最底部 →「Chrome 可以发送通知」

关掉这个之后，Chrome 浏览器的更新提醒、下载完成提示、后台运行提示都会消失。这些消息其实有点用（提醒你更新浏览器），但如果你觉得烦也可以关。

Windows 系统通知

即使 Chrome 的所有通知都关了，Windows 10/11 的通知中心仍然可能在 Chrome 有活动时弹提示。

关闭路径：Windows 设置 → 系统 → 通知 → 找到 Google Chrome → 关掉

macOS 类似：系统设置 → 通知 → Google Chrome → 关掉

这三层关完，你才能真正实现"零弹窗"。大部分教程只讲到第一层（网站通知），这就是为什么很多人按教程关了还是收到弹窗的原因。

[Windows 系统 Chrome 通知设置](/images/tips/chrome-notification-settings/img-3.jpg)

移动端同步：手机 Chrome 怎么关通知

Chrome 的通知设置不会在手机和电脑之间自动同步，需要分别设置。

Android Chrome

菜单（⋮）→ 设置 → 网站设置 → 通知 → 关闭「通知」开关

iOS Chrome

设置 → 内容设置 → 通知 → 关闭「允许通知」

移动端操作更简单，一个开关就全关了。如果你用 Chrome 手机版 的时候经常被通知打扰，建议直接关掉——手机屏幕本来就小，通知弹窗对阅读体验的干扰比电脑端严重得多。

高级技巧：用 Chrome 扩展程序精细控制通知

Chrome 自带的通知管理已经够用，但如果你需要更精细的控制（比如只屏蔽声音不屏蔽通知、按时间段自动关闭通知），可以考虑安装 Chrome 扩展程序：

  • Notification Control：可以对每个网站单独设置通知行为（屏蔽声音、只显示标题、完全屏蔽）
  • Don’t Track Me Google：除了隐私保护，还有通知管理功能

不过说实话，这些扩展程序对大部分用户来说是过度配置。Chrome 自带的三层管理体系（网站通知 → Chrome 提示 → 系统通知）已经覆盖了所有场景。

如果你只是想安静地上网，按下面的「三步清理法」操作就行，不需要装任何额外工具。

[Chrome 通知三步清理法](/images/tips/chrome-notification-settings/img-4.jpg)

三步清理法总结

按这个顺序操作，5 分钟解决 Chrome 通知骚扰问题：

第一步：chrome://settings/content/notifications → 「移除所有站点通知权限」— 清空历史记录

第二步：把你常用的几个工作/安全网站（Gmail、日历等）手动设为允许

第三步：chrome://settings/content/notifications 底部关掉「Chrome 可以发送通知」，Windows 设置里关掉 Chrome 系统通知

三步走完，Chrome 的通知就从"垃圾信息轰炸"变成了"只有你需要时才出现"。如果你后续发现某个网站的通知有用，随时在 chrome://settings/content/notifications 里把它加回允许列表就行。

记住一个原则：通知权限默认应该是屏蔽的，只有当你明确需要某个网站的通知时才单独允许。这比"默认允许、事后一个个关"省心得多。Chrome 通知机制的问题就在于它默认弹询问框——大部分人在没想清楚的情况下就点了"允许"，日积月累通知列表就炸了。

一次性清理干净，以后新网站请求通知时养成"先屏蔽再考虑"的习惯，Chrome 就不会再打扰你了。

常见问题

Q1: 通知设为屏蔽后，网站还能再请求权限吗？

A: 看你是如何设置的。如果你在通知列表里把某个网站设为「屏蔽」，该网站之后无法再弹出通知请求弹窗——Chrome 会静默拦截。但如果你只是把开关设为「默认（询问）」，网站访问时仍然会弹出询问框。如果你用了全局关闭（顶部「网站可以发送通知」开关关掉），所有网站都无法请求，且已授权的也会失效。

Q2: Chrome 通知权限会和 Google 账号同步到其他设备吗？

A: 不会。Chrome 的通知权限设置是设备本地存储的，不会通过 Google 账号同步到其他电脑或手机。这意味着你在公司电脑上允许了某网站的通知，家里的电脑不会自动获得同样的设置。每台设备需要单独配置。好消息是这也意味着你在手机上关掉通知不会影响电脑端的设置。

Q3: Chrome 通知弹窗和 Windows 通知中心是什么关系？

A: Chrome 网站通知通过 Windows 通知中心（Notification Center）来显示弹窗，所以即使 Chrome 本身没在前台，通知也能弹出。这就是为什么关闭了 Chrome 的网站通知权限后，还需要去 Windows 系统设置里单独关闭 Chrome 的系统通知权限——如果只关了一边，另一边还会响。两者是独立开关，需要分别管理。

Q4: 如何让邮件类网站的通知只显示提醒但不弹窗？

A: Chrome 原生不支持「静音通知」功能——每个网站的通知要么允许（弹窗+声音），要么屏蔽（什么都没有）。如果你需要精细控制（比如 Gmail 来邮件时只在标签页上显示小红点但不弹窗），可以借助第三方扩展程序如 Notification Control。或者更直接的做法：把邮件网站的通知关掉，改用手机 Gmail/Outlook App 的推送通知来替代。

Q5: 隐私模式下通知权限会被继承吗？

A: 不会。Chrome 隐私模式默认禁用所有扩展，且不保存会话数据。如果你在正常模式下允许了某些网站的通知权限，在隐私窗口中这些权限不生效——隐私窗口里的通知行为等同于「默认（询问）」。这是隐私模式设计的一部分：保证隐私浏览不留任何持久化痕迹。如果你对隐私模式的运作机制感兴趣，可以看 Chrome 隐私模式详解。
