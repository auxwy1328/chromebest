你登录邮箱、购物网站、社交平台的时候，Chrome 几乎每次都会弹出一个气泡，问你一句：要不要保存密码？大多数人连想都不想就点了保存。毕竟，下次登录时 Chrome 自动帮你填好账号密码，省得自己记。

但你有没有想过，Chrome 到底是怎么知道你在登录的？它保存的密码存在哪里？如果你在网吧或图书馆的电脑上也点了保存，会发生什么？密码同步到 Google 账号之后，Google 能不能看到你的密码？

这些问题很少有人去深究。作为全球使用量最大的浏览器，[Chrome](/) 从 2008 年开始提供密码保存功能，绝大多数用户对它的了解停留在开启和关闭两个按钮上。这篇文章把 Chrome 密码保存的底层机制拆开讲清楚，帮你根据自己的使用场景做出真正合理的设置。

## Chrome 是怎么检测登录表单的

Chrome 能在你登录网站时自动弹出保存提示，背后的检测机制比大多数人想象的要精细。

### 表单识别逻辑

Chrome 通过分析网页的 HTML 结构来判断当前表单是不是一个登录表单。它的检测引擎会关注以下几个关键信号：

**输入类型识别。** Chrome 首先会扫描页面上的 `<input>` 元素，寻找 `type="password"` 的字段。只要页面上存在密码输入框，Chrome 就会高度关注这个表单。接着它会检查同一个 `<form>` 标签内是否存在 `type="text"` 或 `type="email"` 的字段，并将其识别为用户名输入框。

**属性信号识别。** 除了 input type，Chrome 还会读取 HTML 属性来判断字段用途。`autocomplete="username"` 和 `autocomplete="current-password"` 是最明确的信号，网站开发者用这些属性告诉浏览器这个字段的用途。`name="username"`、`name="email"`、`id="login"` 等常见的命名惯例也会被 Chrome 作为辅助判断依据。

![一个登录表单的 HTML 源码截图，高亮标注 type="password"、autocomplete="username" 等关键属性](/images/tips/chrome-save-password/img2.jpg)

**页面上下文分析。** 新版本的 Chrome 还会分析表单所在的页面上下文。如果一个表单出现在页面顶部导航栏区域，且页面 URL 中包含 `/login`、`/signin`、`/auth` 等路径关键词，Chrome 对这个表单是登录表单的置信度会显著提高。反之，如果密码字段出现在注册页面（`/register`、`/signup`）中，Chrome 通常不会触发保存提示。

### 保存触发的时机

Chrome 不是在你打开登录页面时就弹出保存提示，而是在你**成功提交表单之后**才触发的。这个设计很关键。Chrome 需要确认你输入的是有效的登录凭证（用户名和密码的组合），而不是随便填了一些无效信息。表单提交成功后，Chrome 会记录你在这个表单中输入的用户名和密码，然后弹出保存提示。

这个时机选择意味着，如果你在登录页面上填写了用户名和密码但没有点击登录按钮，Chrome 不会提示保存。只有当表单数据真正被提交到服务器的那一刻，Chrome 才会介入。

### 动态网页的处理

现代网站大量使用 JavaScript 来渲染登录表单。很多网站的登录框不是在页面 HTML 源码中写死的，而是通过 JS 动态插入到页面中的。Chrome 使用 MutationObserver API 来监听 DOM 变化，当检测到新的密码输入框被动态添加到页面中时，会重新评估是否需要触发保存逻辑。

这解释了一个常见现象：有些网站的登录弹窗是点击按钮后才出现的，Chrome 依然能正确识别并提示保存密码，即使这个表单在页面初始加载时根本不存在。

## 三种保存策略的区别

Chrome 提供了三种密码保存策略，这个设置藏在 Chrome 设置的密码管理器里，但很多人从来没注意过它的存在。

### 自动保存

这是 Chrome 的默认行为。当你登录一个网站后，Chrome 会自动弹出保存提示气泡。如果你不做任何操作，气泡会在几秒后自动消失，Chrome 会默认保存这个密码。如果你点了一次保存，Chrome 会在后续登录时不再重复询问这个网站的密码（除非密码发生了变化）。

自动保存模式下，Chrome 偶尔也会判断失误。比如你在修改密码页面输入了新密码，Chrome 可能会弹出一个更新密码的提示。大多数情况下这个判断是准确的，但有些网站的修改密码页面的表单结构和登录表单太像，Chrome 会误认为是登录行为。

![Chrome 设置中密码保存策略的三个选项截图，标注当前选中的自动保存选项](/images/tips/chrome-save-password/img3.jpg)

### 询问后保存

和自动保存不同，询问模式下 Chrome 每次检测到登录行为都会弹出提示，**不会自动保存**。你需要手动点击气泡上的"保存"按钮，密码才会被记录。这个模式的优点是你对每个被保存的密码都有明确控制，缺点是操作更繁琐，因为每次登录新网站或密码变更时都要多一步确认。

适合对安全性要求较高、想明确控制 Chrome 密码库内容的用户。如果你发现 Chrome 密码库里积累了一堆你自己都不记得保存过的网站账号，切换到询问模式可以帮你阻止未来的无意识积累。

### 永远不保存

这是最严格的政策。Chrome 完全不会弹出任何保存提示，也不会保存任何新密码。注意，开启这个选项**不会删除已经保存的密码**，只是不再保存新的。如果你想让 Chrome 完全不管理密码，需要先手动清除所有已保存的密码，再开启这个选项。

选择永远不保存的用户通常是使用第三方密码管理器的。如果你安装了 Bitwarden、1Password、KeePass 等专业密码管理工具，完全可以用它们的浏览器扩展来接管密码保存和自动填充功能。关闭 Chrome 自带的保存功能可以避免两个密码管理器之间的冲突，也能消除 Chrome 密码管理的安全隐患。

### 如何切换保存策略

操作步骤很简单：

1. 打开 Chrome，点击右上角三点菜单 → 设置。
2. 在左侧导航栏中找到"密码和自动填充"并点击。
3. 点击"Google 密码管理器"进入密码管理器页面。
4. 在页面顶部找到"提供保存密码"选项。
5. 在三个选项中选择你需要的策略。

![Chrome 密码管理器设置页面，显示提供保存密码的三选项开关，鼠标悬停在询问后保存选项上](/images/tips/chrome-save-password/img4.jpg)

切换策略后立即生效，不需要重启 Chrome。下次你登录任何网站时，Chrome 就会按照新策略来处理密码保存。

## 管理特定网站的密码保存行为

有时候你并不想关闭 Chrome 的全局密码保存功能，但确实有少数几个网站你不希望 Chrome 保存密码。Chrome 为这种需求提供了两种解决方案。

### Never Saved 列表

Chrome 内部维护了一个叫做 Never Saved 的黑名单列表。当你把某个网站加入这个列表后，以后每次登录这个网站，Chrome 都不会再弹出保存密码的提示。

**添加方法：**

1. 打开 Chrome 设置 → 密码和自动填充 → Google 密码管理器。
2. 在密码管理器页面中，找到"从不保存"区域。
3. 点击"添加"按钮，输入你不希望保存密码的网站域名。
4. 点击添加确认。

Chrome 会精确匹配你输入的域名。比如你输入 `bank.example.com`，只有这个子域名的登录不会被保存，`mail.example.com` 仍然会正常弹出保存提示。如果你想屏蔽整个域名下所有子站点，需要在域名前面加一个通配符，或者分别添加每个子域名。

**适用场景：** 银行网站、公司内部系统、你手动管理密码的高安全级别账户。这些场景下，你可能使用独立的密码管理方案，不希望 Chrome 介入。

![Chrome 密码管理器的 Never Saved 列表截图，显示已添加的几个银行域名](/images/tips/chrome-save-password/img5.jpg)

### 临时忽略保存提示

如果你只是这一次不想保存某个密码，不需要把它加入黑名单。当 Chrome 弹出保存提示气泡时：

- 点击气泡上的 × 关闭按钮，Chrome 会跳过这次保存，但下次登录这个网站时仍然会再次询问。
- 点击气泡上的齿轮图标，可以看到"不要为这个网站提供保存密码"的选项，选择后等同于把该网站加入 Never Saved 列表。

这个一次性忽略的功能很实用。比如你在朋友的电脑上临时登录了自己的某个账号，你不想保存密码到朋友的 Chrome 里，关闭气泡即可。下次你回到自己的电脑上，Chrome 仍然会正常提示保存。

### 删除已保存的密码

如果你之前已经保存了某个网站的密码，现在想删除它：

1. 打开 Chrome 设置 → 密码和自动填充 → Google 密码管理器。
2. 在"已保存的密码"列表中找到该网站。
3. 点击条目右侧的删除按钮（垃圾桶图标）。
4. Chrome 会立即删除这条密码记录，不经过确认步骤。

删除后，下次你登录这个网站时 Chrome 会重新弹出保存提示。如果你希望 Chrome 以后都不再保存这个网站的密码，记得在删除后把网站加入 Never Saved 列表。关于 Chrome 密码管理器的更多操作细节，可以参考 [Chrome 查看保存密码完整教程](/tips/chrome-password-view/)。

![Chrome 密码管理器中已保存密码列表的删除操作截图，鼠标悬停在某个网站的删除按钮上](/images/tips/chrome-save-password/img6.jpg)

## 密码同步到 Google 账号的利与弊

Chrome 的密码同步功能是很多用户开启这个浏览器的关键原因之一。你在办公室电脑上保存的密码，回到家打开笔记本上的 Chrome 就能直接自动填充。这种体验确实方便，但你需要理解背后的安全权衡。

### 同步的工作机制

当你登录 Chrome 的 Google 账号并开启密码同步后，Chrome 会将你本地保存的所有密码加密后上传到 Google 服务器。当你在另一台设备上用同一个 Google 账号登录 Chrome 时，密码会被下载并解密到本地。

加密过程大致如下：Chrome 使用你设备上的本地密钥（由操作系统生成和管理）来加密密码明文。加密后的数据通过 TLS 安全通道上传到 Google 服务器存储。在另一台设备上，Chrome 使用那台设备上的本地密钥解密密码。

Google 官方的说法是：Google 服务器上存储的是加密后的密码数据，Google 自己没有解密密钥，因此无法读取你的明文密码。从技术架构来看，这个说法在正常情况下是成立的。

### 开启同步的好处

**跨设备无缝衔接。** 这是最明显的好处。你在手机上保存了一个新密码，电脑上立刻就能用。不需要手动复制粘贴，不需要发消息给自己。对于在多台设备之间频繁切换工作的用户来说，这个体验几乎是不可替代的。

**密码丢失恢复。** 如果你重装了系统或者换了新电脑，只要登录 Google 账号，所有密码都能自动同步回来。相比于手动导出导入密码文件，同步恢复要方便得多。

**泄露检查的全局视角。** Chrome 的密码泄露检查功能需要配合 Google 账号才能使用。开启同步后，Chrome 可以持续监测你所有设备上的密码是否出现在已知的数据泄露中，并在任何一台设备上通知你。

![Chrome 同步设置页面截图，显示密码同步开关已开启，下方列出已同步的设备列表](/images/tips/chrome-save-password/img7.jpg)

### 同步带来的风险

**依赖 Google 的安全承诺。** 虽然技术上 Google 服务器存储的是加密数据，但 Google 有能力修改同步机制。比如更新客户端后改变了加密方案，或者新增了某种服务端解密能力。这意味着你的密码安全性在一定程度上取决于 Google 的商业决策和技术路线，而不是你能控制的。

**攻击面扩大。** 同步意味着你的密码不再只存在于你本地电脑上，而是分布在 Google 的云基础设施中。如果 Google 账号本身被入侵（比如通过钓鱼攻击、凭证泄露），攻击者不仅可以访问你的 Gmail 和 Drive，还可以通过同步获取你在所有设备上的密码。这是一个非常严重的连锁风险。

**数据驻留问题。** Google 的服务器分布在全球各地的数据中心。如果你所在的行业或地区对数据存储位置有合规要求（比如金融行业、医疗行业），密码同步可能不符合合规规定。

### 我的建议

如果你是普通个人用户，只在个人设备上使用 Chrome，密码同步的便利性通常大于风险，可以开启。但请确保你的 Google 账号使用了强密码且开启了两步验证。

如果你处理的是敏感数据（银行、公司内部系统、客户账户），或者你已经在使用 Bitwarden、1Password 等专业密码管理器，建议关闭 Chrome 的密码同步，让专业工具来接管。关于不同密码管理器的对比，可以查看 [Chrome 密码管理器对比](/plugins/chrome-password-manager-comparison/) 这篇文章。

## 共享电脑上的密码安全隐患

这个问题值得单独展开讲，因为很多用户的密码泄露事件恰恰发生在共享电脑上。

### 共享电脑的危险场景

网吧、图书馆、学校机房、公司公共电脑、甚至是借给朋友用的个人电脑，这些场景都属于共享电脑范畴。在共享电脑上，最危险的操作之一就是使用 Chrome 登录自己的账号。

问题不在于你登录后忘记退出——虽然那也是问题——更大的隐患在于 Chrome 的密码保存机制。如果你的 Chrome 账号处于登录状态，或者你允许 Chrome 保存了密码，下一个使用这台电脑的人可以通过多种方式获取你的密码：

**密码管理器直接查看。** 打开 Chrome 设置 → 密码和自动填充 → Google 密码管理器，可以看到所有已保存的密码。虽然查看明文密码需要输入 Windows 登录密码或 PIN 进行验证，但如果上一个用户没有注销 Windows 会话，下一个用户直接就能通过验证。

**自动填充陷阱。** 即使你不主动查看密码管理器，Chrome 的自动填充功能本身就是信息泄露源。当你打开一个网站时，Chrome 如果自动填充了用户名和密码，旁边站着的人就能看到。更狡猾的攻击方式是：攻击者在页面上植入一个隐藏的表单，自动填充的密码会被悄悄发送到攻击者的服务器。

**登录状态劫持。** 如果你登录了某个网站但没有注销，下一个用户打开同一个网站时可能直接进入你的账户。这在社交平台、邮箱等长期保持登录状态的网站上尤其危险。

### 在共享电脑上的正确做法

**使用隐私模式。** [Chrome](/) 的隐私模式不会保存任何密码、Cookie 或浏览记录。关闭隐私窗口后，所有会话数据都会被清除。这是在共享电脑上临时登录自己账号的最佳方式。详见 [Chrome 隐私模式指南](/tips/chrome-incognito-mode/)。

**使用 Chrome 访客模式。** 访客模式创建一个完全独立的浏览会话，和你已登录的 Chrome 配置文件完全隔离。在访客模式下保存的密码不会写入你的主配置文件，访客会话结束后所有数据自动清除。不过要注意，[Chrome 访客模式的](/tips/chrome-guest-mode/) 自动填充功能默认是关闭的，适合在共享电脑上使用。

**关闭密码保存功能。** 如果你必须在共享电脑上使用 Chrome 的普通模式（比如需要使用已保存的书签或扩展），务必在开始操作前关闭密码保存功能。打开 Chrome 设置 → 密码和自动填充 → Google 密码管理器 → 提供保存密码，选择"永远不保存"。使用完毕后，检查并清除浏览器数据，[Chrome 清除缓存和 Cookie](/tips/chrome-clear-cache/) 是必要的清理步骤。

![Chrome 访客模式窗口截图，显示地址栏中的访客图标和独立的标签页界面](/images/tips/chrome-save-password/img8.jpg)

### 公司电脑的特殊注意事项

很多人低估了公司电脑的风险。你可能觉得公司电脑是你一个人在用，但在 IT 管理层面，情况完全不同：

- 公司 IT 部门通常有管理员权限，可以访问你电脑上的文件，包括 Chrome 的密码数据库。
- 企业版 Chrome 可能被配置了密码导出策略，管理员可以批量导出员工保存的密码。
- 公司电脑通常安装了终端管理软件（DLP 系统），可能会监控或拦截敏感数据的输入和传输。

在公司电脑上，最安全的做法是使用公司指定的密码管理工具（通常是企业版 1Password 或 Okta），而不是依赖 Chrome 自带的密码保存功能。如果你用 Chrome 保存了个人账号的密码，这些密码在公司 IT 管理员面前几乎没有秘密可言。

## Chrome 密码保存与第三方密码管理器的共存

如果你已经决定使用第三方密码管理器，如何处理 Chrome 自带的保存功能是一个需要认真考虑的问题。

### 两套系统并行的问题

Chrome 和第三方密码管理器（Bitwarden、1Password、KeePass 等）同时开启密码保存功能时，会出现几个实际问题：

**弹窗冲突。** 你登录一个网站时，Chrome 弹出保存提示的同时，密码管理器的扩展也可能弹出自己的保存提示。两个气泡同时出现，互相遮挡，不仅影响操作体验，还容易导致误操作。

**自动填充竞争。** 当你打开一个登录页面时，Chrome 的自动填充和密码管理器的自动填充可能会同时触发。结果可能是两个下拉列表互相覆盖，你填入的是 Chrome 记录的密码还是密码管理器的密码，取决于哪个扩展先执行了填充操作。这会导致混乱，尤其是在你同时更新了密码但只在其中一个管理器中更新了的情况下。

![一个登录页面上同时出现 Chrome 密码保存气泡和 Bitwarden 扩展保存气泡的截图，展示两个气泡互相遮挡的情况](/images/tips/chrome-save-password/img9.jpg)

### 推荐的共存方案

根据你的实际需求，有以下几种方案：

**方案一：关闭 Chrome 保存，完全使用第三方管理器。** 这是最干净的方案。在 Chrome 设置中把提供保存密码设为"永远不保存"，同时在 Chrome 的自动填充设置中关闭密码自动填充。所有密码保存和填充都交给第三方管理器的浏览器扩展处理。适合已经全面迁移到第三方管理器的用户。

**方案二：Chrome 保存但关闭自动填充，第三方负责填充。** 这个方案适合你在不同设备上使用不同密码管理工具的情况。Chrome 只负责存储密码（作为备份），但不自动填充。自动填充完全由第三方管理器的扩展来执行。这种方案的问题是 Chrome 和第三方管理器各存一份密码，更新密码时需要两处都改。

**方案三：只在部分场景使用 Chrome 保存。** 比如你不重要的个人账号用 Chrome 保存，银行和公司账号用 Bitwarden 管理。通过 Chrome 的 Never Saved 列表把敏感网站排除在外。这种方案比较实用，但需要注意 Chrome 密码库和 Bitwarden 密码库之间的维护成本。

如果你还没有决定用哪个密码管理器，[Chrome 必备扩展推荐](/plugins/chrome-essential-extensions/) 中对几款主流密码管理器扩展有简要介绍。而 [Chrome vs Brave 浏览器对比](/compare/chrome-vs-brave-2026/) 中也提到，Brave 内置的密码管理器在隐私保护方面和 Chrome 有明显区别，如果你对密码隐私特别在意，可以了解一下 Brave 的方案。

## 密码保存相关的常见问题解答

### Chrome 保存密码时会存储哪些信息

Chrome 保存一条密码记录时，存储的信息包括：网站域名（URL 的 origin 部分）、用户名（或邮箱地址）、密码（加密存储）、以及保存时间戳。Chrome 不会保存你输入的其他表单字段（比如验证码、记住我勾选框等）。

### 为什么 Chrome 有时候不提示保存密码

这种情况比较常见，可能的原因包括：

- 网站使用了自定义的登录表单（非标准 HTML form 结构），Chrome 无法正确识别。
- 网站通过 JavaScript 拦截了表单提交事件，Chrome 的表单提交监听器没有被触发。
- 网站设置了 `autocomplete="off"` 属性，Chrome 会尊重这个设置（虽然新版 Chrome 已经开始忽略这个属性）。
- 你在隐私模式或访客模式下登录，这些模式默认不保存密码。
- 该网站已经在你的 Never Saved 列表中。

### 重装 Chrome 后密码还在吗

如果你开启了 Google 账号同步，重装 Chrome 后登录同一个 Google 账号，密码会自动同步回来。如果没有开启同步，重装 Chrome 会清空本地密码数据库，密码将无法恢复。这也是为什么建议开启同步或定期导出密码备份的原因。

### Chrome 保存的密码会被恶意软件窃取吗

这是一个真实存在的威胁。恶意软件如果以你的用户权限运行，可以通过以下方式窃取 Chrome 密码：

- 直接读取 Chrome 的 `Login Data` SQLite 数据库文件，配合 Windows DPAPI 解密。
- 使用键盘记录器在你输入 Windows 验证密码时截获密钥。
- 通过浏览器扩展注入脚本来捕获自动填充的密码。

防范措施包括：不在不可信的来源下载安装软件、保持杀毒软件更新、定期检查已安装的 [Chrome 扩展](/) 列表。关于 Chrome 安全设置的全面指南，可以参考 [Chrome 隐私安全设置详解](/tips/chrome-incognito-mode/)。

### 多个 Chrome 配置文件各自独立保存密码吗

是的。Chrome 的密码数据库是按用户配置文件隔离的。如果你在 Chrome 中创建了多个配置文件（比如工作配置文件和个人配置文件），每个配置文件有自己独立的密码库。配置文件 A 中保存的密码不会出现在配置文件 B 的密码管理器中，反之亦然。但如果你在多个配置文件中都登录了同一个 Google 账号并开启了同步，密码会被同步到所有配置文件。

![Chrome 配置文件切换界面截图，显示多个用户头像，每个配置文件代表独立的密码存储空间](/images/tips/chrome-save-password/img10.jpg)

## 根据你的场景选择最佳设置

说了这么多机制和风险，最后给你一个实用的设置建议，按使用场景分类：

**个人电脑，只用 Chrome：** 开启自动保存 + 开启密码同步 + 开启泄露检查。这是最方便的配置，安全性在个人设备上足够。

**个人电脑，Chrome + 第三方密码管理器：** Chrome 设为永远不保存 + 关闭自动填充。所有密码管理交给第三方工具。Chrome 的密码功能完全禁用，避免冲突。

**公司电脑：** Chrome 设为永远不保存 + 使用公司指定的密码管理工具。不要在 Chrome 中保存任何个人或工作密码。

**共享电脑（网吧、图书馆）：** 使用隐私模式或访客模式。不要在共享电脑上使用已登录的 Chrome 配置文件，不要保存任何密码。

**高安全需求（金融、安全从业者）：** Chrome 永远不保存 + 使用硬件安全密钥（YubiKey 等）配合专业密码管理器。所有密码通过硬件密钥二次验证才能访问。

Chrome 的密码保存功能本身没有对错，关键在于你是否理解它的工作方式，并根据自己真实的使用场景做出合理的配置。大多数人只需要花两分钟检查一下自己的设置，就能显著提升密码管理的安全性和可控性。

## 常见问题

### Q1: Chrome 保存密码时提示「密码已泄露」是不是误报？

A: 有可能，但不能直接假设是误报。Chrome 的泄露检查是将你密码的哈希值与 Google 维护的已知泄露数据库进行比对，如果匹配说明该密码确实出现在过某次公开的数据泄露事件中。有可能是同一个密码被其他用户使用后泄露了，而你只是恰好用了相同的密码。但无论如何，收到这个提示就意味着这个密码已经暴露在公开渠道中，建议立即更换。即使不是你的账号被泄露，攻击者也会用泄露数据库中出现的密码尝试登录各种网站。

### Q2: 在隐私模式下登录网站，Chrome 还会提示保存密码吗？

A: 不会。隐私模式默认不提示保存密码、不自动填充密码，也不在密码管理器中记录任何新保存的密码。这是隐私模式设计的一部分。如果你在隐私窗口中登录了某个网站并希望 Chrome 记住密码，必须在正常模式下重新登录一次。详情参考 [Chrome 隐私模式详解](/tips/chrome-incognito-mode/)。

### Q3: Chrome 保存的密码可以迁移到其他浏览器吗？

A: 可以。最直接的方法是通过 CSV 导出（在 Chrome 密码管理器中选择「导出密码」），然后在目标浏览器（Firefox、Edge 等）的密码管理器中导入 CSV 文件。Edge 支持直接从 Chrome 导入密码（不需要导出 CSV），Firefox 也支持从 Chrome 导入。注意 CSV 导出文件中的密码是明文的，迁移完成后立即删除。如果你想迁移到专业的密码管理器如 Bitwarden，它支持直接读取 Chrome 的密码数据库而不需要导出 CSV。

### Q4: 为什么某些网站的密码总是保存不上？

A: 通常是因为网站使用了非标准登录表单（JavaScript 动态生成的表单、单页应用的登录弹窗），Chrome 无法正确识别表单提交事件。你可以尝试：先手动在密码管理器中添加该网站的登录凭据；或者在登录时确保用户名和密码都完整填写后，先点击页面其他位置让表单验证完成，再提交。如果还不行，可以借助密码管理器扩展（Bitwarden、1Password）的「手动保存」功能来绕过这个限制。

### Q5: Chrome 和 Google 账号的密码是同一个概念吗？

A: 不是。Chrome 保存的是你在各个网站上的登录密码（如淘宝、微博、GitHub），而 Google 账号密码是你用来登录 Google 服务（Gmail、YouTube、Chrome 同步）的那个主密码。两者完全独立。Chrome 的密码同步功能会把你在各网站保存的密码通过 Google 账号同步到其他设备，但同步过程中这些网站密码是用设备本地密钥加密的，Google 声称无法解密你的网站密码。不过你的 Google 账号密码本身的安全至关重要——如果它被破解，攻击者可以访问你的 Gmail、Drive，并可能通过重置其他网站密码来间接获取更多账户的访问权。
