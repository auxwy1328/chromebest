上周帮同事排查一个网页显示错乱的问题——他坚持说自己没改过任何设置，结果一看是缓存里存了旧版 CSS，新代码上线后浏览器还在用旧样式。清完缓存立刻好了。

Chrome 缓存这东西，不清会出 bug，全清会退出登录，清错了还可能丢数据。这篇把三种清理方式讲清楚，以及什么时候该用哪种。

![chrome-clear-cache](/images/tips/chrome-clear-cache/body1.jpg)

## Chrome 缓存是什么，为什么会出问题

缓存是 Chrome 把你访问过的网页资源（图片、CSS、JS 脚本）存在本地硬盘上，下次访问同一网页时直接从硬盘读取，不用重新下载。

这个机制大部分时候很好用，但会出问题：

- **网页更新了但显示旧内容**：开发者改了代码，你的浏览器还在用缓存的旧文件
- **登录状态异常**：缓存了旧的 Session 数据，导致登录态不一致
- **缓存损坏**：Chrome 异常关闭后缓存文件损坏，导致页面加载失败或显示错乱
- **缓存太大占磁盘**：长期不清理，缓存能占到好几个 GB

## 方法一：手动清除缓存（最常用）

### 标准步骤

1. 点击 Chrome 右上角 **⋮** → 「设置」
2. 左侧选择「隐私与安全」→ 「清除浏览数据」
3. 或者直接按 **Ctrl+Shift+Delete**（Mac: **Cmd+Shift+Delete**）

### 关键选项说明

弹出的窗口里有三个关键选项：

| 选项 | 清除什么 | 建议是否勾选 |
|------|----------|-------------|
| 浏览记录 | 你访问过的网页地址 | ❌ 不建议清 |
| Cookie 和网站数据 | 登录状态、网站偏好 | ⚠️ 看情况 |
| 缓存的图片和文件 | 网页临时资源文件 | ✅ 必须勾选 |

**如果你想解决网页显示异常**：只勾选「缓存的图片和文件」，时间范围选「过去 7 天」。

**如果你要彻底清理**：三个都勾，但做好重新登录所有网站的准备。

### 时间范围选择

- **过去 1 小时**：刚出问题时用，影响最小
- **过去 24 小时**：日常问题排查
- **过去 7 天**：不确定什么时候出的问题
- **时间不限**：核弹级，清空所有数据

绝大多数情况下选「过去 7 天」就够用了。

## 方法二：只清除指定网站的缓存（最精准）

全清缓存会退出所有网站登录，很烦。如果你只是某个网站显示异常，用这个方法：

### 步骤

1. 打开有问题的网站
2. 按 **F12** 打开开发者工具
3. 右键点击地址栏左边的**刷新按钮**（🔄）
4. 选择「**清空缓存并硬性重新加载**」

这个方法只清除当前网站的缓存，其他网站的登录状态完全不受影响。也是我日常用得最多的方法。

### 另一种方式：通过站点设置

1. 点击地址栏左边的锁头图标（或 ⓘ 图标）
2. 点击「网站设置」
3. 点击「清除数据」
4. 只勾选「缓存的图片和文件」→ 清除

这种方式更直观，但步骤多一些，适合不想按 F12 的用户。

## 方法三：自动清理缓存

Chrome 本身不支持自动清理，但有两种方式实现。

### 方式 A：限制缓存大小

在 Chrome 快捷方式的目标路径后面加启动参数：

```
--disk-cache-size=104857600
```

这会把缓存限制在 100MB（默认无上限）。步骤：

1. 右键桌面 Chrome 图标→「属性」
2. 「目标」框最后加一个空格，粘贴上面的参数
3. 确定后重启 Chrome

100MB 对日常浏览够用，超出的旧缓存会被自动淘汰。

### 方式 B：用扩展程序自动清理

推荐 **Click&Clean** 扩展：

1. 安装后在扩展选项里设置「关闭浏览器时自动清除」
2. 只勾选「缓存」，不要勾选 Cookie 和密码
3. 每次关闭 Chrome 自动清理缓存，不影响登录状态

设置好之后完全不用手动管，适合怕麻烦的用户。

## 查看缓存占用了多少空间

### Chrome 内置查看

1. 地址栏输入 `chrome://settings/siteData`
2. 可以看到每个网站的存储用量
3. 按占用大小排序，找出最占空间的网站

### 系统级查看

Windows 下缓存文件位置：

```
C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data\Default\Cache
```

右键这个文件夹→「属性」可以看到总大小。我见过最夸张的占了 8GB，一般 1-3GB 比较常见。

**不要直接删这个文件夹**，可能导致 Chrome 启动异常。用浏览器内置功能清理最安全。

## 清理缓存后常见问题

### 清理后所有网站都退出登录了

原因：清理时勾选了 Cookie。解决：
1. 下次清理只勾选「缓存」不勾选 Cookie
2. 如果已经清了，只能重新登录，没有恢复办法

### 清理后网页加载变慢

正常现象。缓存没了，每个网页都要重新下载所有资源。访问 2-3 次后速度会恢复正常。

### 清理后某个网站还是显示旧内容

可能是网站用了 CDN 缓存，不只是浏览器缓存的问题。试试：
1. 强制刷新：**Ctrl+F5**
2. 或者用无痕模式打开同一网页，如果无痕模式显示正常，说明浏览器缓存没清干净
3. 清理 DNS 缓存：命令行执行 `ipconfig /flushdns`

### 清理后 Chrome 变卡

极少数情况下清理大量缓存后 Chrome 会短暂变慢，因为内部索引在重建。关掉 Chrome 等 30 秒再重新打开就好了。关于更多 Chrome 卡顿问题可以参考[Chrome 内存占用优化教程](/tips/chrome-memory-optimization/)。如果清理缓存后遇到页面渲染异常（花屏、黑屏），也可能是[Chrome 硬件加速设置](/tips/chrome-hardware-acceleration/)的问题，二者有时会被混淆。

### 清理后视频网站播放异常

清理缓存时如果同时清了 Cookie，视频网站的播放记录和偏好会丢失，但不会影响播放功能本身。如果出现视频无法播放，检查是否误清了 Flash 或媒体许可证数据。如果更新后也有问题，可以参考[Chrome 更新失败修复方法](/tips/chrome-update-failed/)。此外，定期清理缓存有助于降低 Chrome 的磁盘占用，和[设为默认浏览器](/tips/chrome-default-browser/)一样属于基础维护操作，建议养成习惯。

## 手机版 Chrome 清理缓存

手机 Chrome 的清理入口和电脑版不同。

### Android 版

1. 打开 Chrome → 右上角 **⋮** → 「设置」
2. 「隐私与安全」→ 「清除浏览数据」
3. 选择时间范围和要清除的数据类型
4. 点击「清除数据」

### iOS 版

1. 打开 Chrome → 右下角 **⋯** → 「设置」
2. 「隐私」→ 「清除浏览数据」
3. 选择要清除的项目 → 「清除浏览数据」

手机版和电脑版的区别：手机版没有 F12 开发者工具，无法单独清除某个网站的缓存。如果只是某个网站有问题，只能全清或者卸载重装 Chrome App。

关于手机版更多技巧可以看[Chrome 手机版使用指南](/tips/chrome-mobile-tips/)。如果清理缓存是为了解决扩展程序异常的问题，也可以先参考[Chrome 扩展不工作的排查方法](/tips/chrome-extensions-not-working/)，有时缓存只是症状而非根因。

## 缓存与隐私：清除缓存能保护隐私吗

很多人以为清除缓存等于保护隐私，实际上缓存和隐私是两个层面的问题。缓存文件本身不会直接暴露你的浏览历史——缓存的图片、CSS、JS 文件命名通常是哈希值，普通人看不懂。但如果你和别人共用一台电脑，以下几种情况缓存会间接暴露隐私：

- **网站图标缓存（Favicon Cache）**：Chrome 会把访问过的网站图标单独存一份。即使清了历史记录，这些图标文件仍然在磁盘上，懂行的人可以通过图标文件名反推你访问过哪些网站
- **预加载缓存**：Chrome 在你输入网址时就会提前加载预测的网页资源。这些页面你可能根本没打开过，但缓存里已经有了它们的文件
- **Service Worker 缓存**：一些网站（如视频站、新闻站）会在浏览器后台存储离线数据。普通的清除缓存操作清不掉 Service Worker 的存储，需要进入 `chrome://settings/siteData` 手动找到对应网站并删除

如果你对隐私有较高要求，建议同时做三件事：清除缓存 + 清除 Cookie + 清除站点存储。只清缓存不碰 Cookie 和站点数据，隐私保护是不完整的。另外，如果你用 Chrome 登录了 Google 账号且开启了同步，清本地缓存后云端仍然可能保留相关数据。关于更全面的隐私设置可以参考 [Chrome 隐私设置完全指南](/tips/chrome-privacy-settings-guide/)。

## 开发者实用技巧：如何在调试时完全绕过缓存

如果你是前端开发、测试人员或者经常需要查看网页最新版本，Chrome 有几招比手动清缓存更高效的方法：

### DevTools 禁用缓存

1. 按 **F12** 打开开发者工具
2. 点击「**Network（网络）**」面板
3. 勾选「**Disable cache（禁用缓存）**」
4. 这个设置只在 DevTools **打开状态**下生效，关闭 DevTools 后缓存恢复正常

这是开发调试时最推荐的设置——开发时禁用缓存看到最新代码，日常浏览时缓存正常工作保证速度。

### 强制刷新快捷键

- **Ctrl+F5**（Mac: **Cmd+Shift+R**）：强制刷新当前页面，跳过缓存重新请求所有资源
- **Ctrl+Shift+Delete** → 只勾选缓存 → 选择「过去 1 小时」：最快捷的手动清理

### 用无痕模式验证是否是缓存问题

如果你不确定问题出在浏览器缓存还是 CDN 缓存，用无痕模式（**Ctrl+Shift+N**）打开同一页面。无痕模式不使用任何已缓存的资源，加载的是服务器最新返回的内容：

- 无痕模式显示正常、普通模式显示旧内容 → 浏览器缓存问题
- 两者都显示旧内容 → 服务器端或 CDN 缓存没更新，请联系网站管理员

这个二分法在排查网页显示问题时非常实用，省去了反复清缓存的麻烦。关于 DevTools 的更多用法可以参考 [Chrome 开发者工具入门指南](/tips/chrome-devtools-beginner-guide/)。
