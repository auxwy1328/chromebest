你每天用 Chrome 浏览网页，登录邮箱、社交平台、银行、购物网站，Chrome 一路帮你记住所有密码。几年下来，你可能已经在 Chrome 里存了几百个账号密码，但你从来没有检查过这些密码的状态。

问题来了：你 Chrome 里到底存了多少密码？其中有多少已经在数据泄露中被公开了？有多少是重复使用的弱密码？Chrome 把你的密码存在本地，安全性到底如何？

这篇文章会带你一步步查看 Chrome 保存的所有密码，教你如何导出密码、检查泄露、评估风险，并分析 Chrome 自带密码管理器的真实安全水平，最后给出是否应该换用专业密码管理器的建议。

## 为什么你需要关心 Chrome 保存的密码

作为全球使用量最大的浏览器，[Chrome浏览器](/) 从 2008 年开始提供密码保存功能，到 2026 年，这个功能已经运行了将近 18 年。根据 Google 的数据，Chrome 密码管理器帮助用户管理着超过数十亿个密码。但对大多数用户来说，密码保存是一个设置一次就再也不管的功能。

这种「设置即遗忘」的习惯带来几个隐患：

**第一，密码复用问题。** Chrome 会保存你第一次登录某个网站时输入的密码，如果多个网站用的是同一个密码，Chrome 也会忠实地保存多份相同的密码。一旦其中一个网站发生数据泄露，攻击者拿到的密码可以用来登录你所有使用相同密码的网站。根据安全研究机构的数据，超过 65% 的用户在不同网站复用密码。

**第二，过期密码积累。** 你可能几年前改了某个网站的密码，但 Chrome 里仍然保存着旧密码。Chrome 虽然会提示保存新密码，但旧密码记录不会自动删除。这意味着你的密码库里存在大量已经失效的密码，增加了混乱和潜在风险。

**第三，安全意识缺失。** 大多数用户从未打开过 Chrome 的密码管理界面，不知道自己存了多少密码，更不知道哪些密码已经在公开的数据泄露中被曝光。Chrome 实际上提供了密码泄露检查功能，但很多人不知道它的存在。

[Chrome密码管理](/) 的核心目的就是方便，但方便和安全之间需要平衡。在开始查看之前，我们先聊聊 Chrome 密码管理器的工作原理。

![Chrome 密码管理器概览截图，显示已保存密码列表](/images/tips/chrome-password-view/img1.jpg)

## Chrome 密码管理器的工作原理

理解 Chrome 如何存储密码，有助于你评估它的安全性并做出正确的管理决策。

Chrome 在本地将密码保存在一个名为 `Login Data` 的 SQLite 数据库文件中。这个文件的路径因操作系统不同而有所区别：

- **Windows**: `C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data\Default\Login Data`
- **macOS**: `~/Library/Application Support/Google/Chrome/Default/Login Data`
- **Linux**: `~/.config/google-chrome/Default/Login Data`

密码本身不是以明文存储的。Chrome 使用操作系统的原生加密机制来保护密码：

- **Windows** 上使用 DPAPI（Data Protection API），密码的加密密钥绑定到你的 Windows 用户账户。这意味着只有当前登录的 Windows 用户才能解密这些密码。
- **macOS** 上使用 Keychain，Chrome 将密码存储在系统钥匙串中，由 macOS 统一管理。
- **Linux** 上使用 GNOME Keyring 或 KDE KWallet，取决于你的桌面环境。

这种设计的一个关键含义是：如果你的电脑登录账户有强密码，且你的电脑没有被恶意软件或物理入侵，Chrome 本地存储的密码是相对安全的。但如果你离开电脑时没有锁屏，或者电脑被安装了恶意软件，本地加密的保护就可能被绕过。

如果你开启了 Chrome 的密码同步功能，密码会通过你的 Google 账号同步到 Google 服务器。同步传输使用 TLS 加密，存储在 Google 服务器上的密码也会被加密。Google 使用你登录设备上生成的密钥来加密同步的密码，这意味着 Google 自己也无法直接读取你的明文密码。

## 如何查看 Chrome 保存的所有密码

### 方法一：通过 Chrome 设置查看

这是最常用的方法，适合大多数用户。

**步骤如下：**

1. 打开 Chrome 浏览器，点击右上角的三点菜单按钮（或直接按快捷键 `Alt + F`）。

![Chrome 右上角三点菜单截图](/images/tips/chrome-password-view/img2.jpg)

2. 在下拉菜单中选择「设置」，进入 Chrome 设置页面。

3. 在设置页面的左侧导航栏中，找到「密码和自动填充」选项并点击。

4. 点击「Google 密码管理器」，会打开密码管理的完整界面。

![Chrome 密码和自动填充设置页面截图](/images/tips/chrome-password-view/img3.jpg)

5. 在密码管理器页面中，你可以看到 Chrome 保存的所有网站、用户名和密码。密码默认被隐藏，显示为一排圆点。

6. 点击某个网站条目，展开详细信息，然后点击密码栏旁边的「眼睛」图标来显示明文密码。

7. Chrome 会弹出系统验证窗口，要求你输入电脑的登录密码或 PIN 码。验证通过后，明文密码就会显示出来。

![点击眼睛图标显示明文密码的截图](/images/tips/chrome-password-view/img4.jpg)

### 方法二：通过 Chrome 地址栏快速访问

如果你经常需要查看密码，可以直接在 Chrome 地址栏输入以下地址：

```
chrome://password-manager/passwords
```

这会直接跳转到密码管理器页面，省去了从设置中逐级点击的步骤。这个方法在 [Chrome 使用技巧](/) 中非常实用，建议记下来。

### 方法三：通过 Google 密码管理器网页版

如果你需要在手机或其他设备上查看 Chrome 保存的密码，可以访问 Google 密码管理器的网页版：

```
https://passwords.google.com
```

这个页面需要你登录 Google 账号，登录后可以看到与 Chrome 同步的所有密码。网页版的布局和 Chrome 内置的密码管理器基本一致，但操作体验在移动设备上更好。

需要注意的是，网页版显示明文密码时，同样需要进行身份验证（输入 Google 账号密码或使用已验证的设备确认）。

### 方法四：通过 Chrome 自动填充提示查看

当你在某个网站上点击登录框时，如果 Chrome 保存了该网站的密码，会弹出自动填充建议。点击这个建议后，Chrome 可能会要求你验证身份（输入电脑密码），然后自动填充用户名和密码。

这个方法不适合批量查看密码，但当你只需要找回某个特定网站的密码时，是最快的方式。

## Chrome 密码泄露检查：找到不安全的密码

Chrome 内置了密码泄露检查功能，这是很多人不知道但非常有用的特性。它的原理是将你保存的密码与 Google 维护的已知数据泄露数据库进行比对。

**使用方法：**

1. 打开 Chrome 密码管理器（chrome://password-manager/passwords）。
2. 页面顶部会显示一个「检查密码」按钮。

![Chrome 密码泄露检查页面截图，显示检查密码按钮](/images/tips/chrome-password-view/img5.jpg)

3. 点击「检查密码」，Chrome 开始与 Google 的泄露数据库进行比对。

4. 检查完成后，Chrome 会列出所有存在问题的密码，分为三类：
   - **已泄露的密码**：这些密码出现在已知的数据泄露事件中，需要立即更换。
   - **重复使用的密码**：同一个密码被用于多个网站，如果一个网站被攻破，其他网站也会受影响。
   - **弱密码**：密码强度不够，容易被暴力破解。

![Chrome 密码检查结果截图，显示泄露、重复、弱密码分类](/images/tips/chrome-password-view/img6.jpg)

**技术原理：** Chrome 的泄露检查并非将你的明文密码发送给 Google。它使用了一种称为「密码哈希前缀匹配」的技术：Chrome 将你密码的哈希值的前几位（前缀）发送给 Google，Google 返回所有匹配此前缀的泄露密码哈希，然后 Chrome 在本地完成完整比对。这意味着 Google 永远看不到你的完整密码。

**需要注意的是：** 密码泄露检查功能需要你登录 Google 账号。如果你使用的是未登录的 Chrome 配置文件，这个功能不可用。

检查结果出来后，建议你按照以下优先级处理：
1. 先处理标记为「已泄露」的密码，这些是最紧急的
2. 再处理重复使用的密码，为每个网站设置独立的密码
3. 最后处理弱密码，将其替换为更强的密码

如果你担心 [Chrome 浏览器的隐私](/tips/chrome-incognito-mode/) 问题，可以选择在检查完成后暂时关闭密码同步功能。

## 如何导出 Chrome 保存的密码

导出密码是迁移到其他浏览器或密码管理器时最常用的操作。Chrome 支持将所有保存的密码导出为 CSV 文件，但这个过程有一个需要特别注意的安全风险。

### 导出步骤

1. 打开 Chrome 密码管理器（chrome://password-manager/passwords）。
2. 点击页面上的齿轮设置图标（或「设置」按钮）。
3. 在弹出菜单中选择「导出密码」。

![Chrome 导出密码菜单截图](/images/tips/chrome-password-view/img7.jpg)

4. Chrome 会弹出警告提示，告知你导出的密码将以明文形式保存在 CSV 文件中。
5. 点击「导出密码」确认。
6. 选择保存位置，输入文件名，点击保存。

![Chrome 导出密码警告提示截图](/images/tips/chrome-password-view/img8.jpg)

导出完成后，你会得到一个 CSV 文件，包含以下字段：网站名称、网址、用户名、密码。这个文件可以用 Excel 或任何文本编辑器打开。

### CSV 明文密码的严重风险

这里需要特别强调一个安全问题：**导出的 CSV 文件中的密码是完全没有加密的明文。**

这意味着什么？

- 如果这个文件存储在你的电脑上，任何能访问你电脑的人（包括恶意软件）都能直接读取所有密码。
- 如果你把这个文件通过邮件发送给任何人（即使是你自己发送到另一个邮箱），邮件传输过程中密码都是明文的。
- 如果你把文件上传到网盘或云存储，一旦账户被盗或存储被入侵，所有密码都会暴露。
- 文件复制到 U 盘后，U 盘丢失就意味着密码泄露。

**安全建议：**

- 导出密码后，**立即使用完毕**。完成密码迁移后，立即删除 CSV 文件。
- **不要通过邮件发送**这个文件。
- **不要长期保存在电脑上**。如果必须保存，使用加密工具（如 BitLocker 加密的文件夹或 VeraCrypt 加密容器）来存放。
- 使用「安全删除」工具（而不是直接删除到回收站）来彻底擦除 CSV 文件。

### 不导出 CSV 的密码迁移方式

如果你只是想把密码迁移到另一个密码管理器，其实不一定要导出 CSV。许多密码管理器支持直接从 Chrome 导入密码，无需经过中间文件。比如 Bitwarden 和 1Password 都可以直接读取 Chrome 的密码数据库来导入，这样密码不会以明文形式出现在磁盘上。

如果你正在考虑 [Chrome 插件推荐](/plugins/chrome-essential-extensions/)，不妨也看看密码管理器相关的插件。

## 密码管理器对比：Chrome 自带 vs Bitwarden vs 1Password vs KeePass

理解不同密码管理器的优缺点，可以帮助你做出最适合自己需求的选择。

### Chrome 内置密码管理器

**优势：**
- 零配置，安装 Chrome 即可用
- 自动保存和自动填充无缝集成
- 跨设备同步（通过 Google 账号）
- 内置密码泄露检查
- 免费，无需额外安装

**劣势：**
- 只能在 Chrome 浏览器内使用，不支持 Firefox、Edge、Safari 等其他浏览器
- 没有安全备注功能
- 没有文件夹/标签分类功能
- 没有内置 TOTP 验证码生成功能
- 没有密码分享功能（家庭团队共用）
- 密码生成器的自定义选项有限
- 导出功能只有 CSV 明文，无加密导出选项
- 开源透明度不如独立密码管理器

### Bitwarden

**优势：**
- 完全开源，代码可审计
- 免费版功能已经非常强大
- 支持所有主流浏览器（Chrome、Firefox、Edge、Safari）和操作系统（Windows、macOS、Linux、iOS、Android）
- 内置 TOTP 验证码生成
- 支持安全备注、文件夹分类
- 支持密码分享（家庭版 1 人免费，团队付费）
- 支持多种导入格式，可以直接从 Chrome 导入
- 支持自托管服务器（完全自主控制数据）
- 端到端加密

**劣势：**
- 免费版的密码分享功能有限（只能 1 人）
- 界面设计相对简洁，不如 1Password 精致
- 需要额外安装浏览器插件和桌面应用
- 自托管需要一定的技术能力

### 1Password

**优势：**
- 行业内公认的最佳用户体验
- Watchtower 功能（类似 Chrome 的泄露检查但更强大，还包括网站安全问题监控）
- 支持 Travel Mode（旅行模式，临时隐藏敏感数据）
- 强大的密码分享和家庭管理功能
- 支持 TOTP、安全备注、自定义字段
- 支持信用卡和身份信息的存储与自动填充
- 支持 Vault（保险库）功能，不同类型数据分开管理

**劣势：**
- 付费（个人版约 $2.99/月，家庭版约 $4.99/月）
- 不开源（代码不可审计）
- 需要订阅制，取消订阅后功能受限

### KeePass

**优势：**
- 完全开源，社区活跃
- 完全本地化，密码文件由你自己控制
- 支持多种加密算法（AES-256、ChaCha20 等）
- 数据库文件可以放在任何地方（本地、网盘、U 盘）
- 无需注册任何账号
- 完全免费，无任何功能限制

**劣势：**
- 自动填充需要第三方插件（如 KeePassXC 或浏览器插件）
- 不内置泄露检查功能
- 跨设备同步需要自己配置（同步数据库文件到云盘）
- 界面相对老旧，学习曲线较高
- 移动端体验不如基于云的方案

如果你已经用过 [Chrome 密码管理器对比](/plugins/chrome-password-manager-comparison/) 相关的评测，应该对这些差异有所了解。核心结论是：Chrome 自带方案适合轻度用户，如果你有超过 50 个密码且关注安全性，Bitwarden（免费）或 1Password（付费）是更值得的选择。

## 如何从 Chrome 迁移密码到第三方管理器

如果你决定换用 Bitwarden 或 1Password，迁移过程其实很简单。以 Bitwarden 为例：

1. 安装 Bitwarden 浏览器插件和桌面应用。
2. 注册 Bitwarden 账号（或选择自托管）。
3. 打开 Bitwarden，进入「工具」→「导入数据」。
4. 数据源选择「Google Chrome」。
5. Bitwarden 会自动读取 Chrome 本地的密码数据库并导入。

![Bitwarden 从 Chrome 导入密码的界面截图](/images/tips/chrome-password-view/img9.jpg)

整个过程不需要导出 CSV，密码不会以明文形式写入磁盘。迁移完成后，建议在 Chrome 中关闭密码保存功能，避免两个密码管理器同时运行造成混乱。

关于关闭 Chrome 密码保存的方法，可以参考我们的 [Chrome 密码保存设置](/tips/chrome-save-password/) 文章。

## Chrome 密码相关的常见问题和解决方案

### 忘记了某个网站的密码，但 Chrome 也没保存怎么办

如果你在某个网站上点击「永不保存此网站的密码」，Chrome 就不会再保存这个网站的密码。要取消这个设置：

1. 打开 Chrome 密码管理器。
2. 滚动到底部的「从不保存」列表。
3. 找到对应网站，点击旁边的删除按钮。
4. 重新登录该网站，Chrome 会再次弹出保存密码的提示。

### 同步后密码不一致

如果你在多台电脑上使用 Chrome，密码同步可能会出现延迟或冲突。Chrome 通常会保留最新的密码版本，但在极少数情况下可能会出现不一致。解决办法是在密码管理器中手动检查并更新有冲突的密码条目。

### 电脑被共享，如何保护 Chrome 中的密码

如果你的电脑经常被其他人使用，有几个保护措施：

- **使用独立的 Windows 用户账户**。Chrome 的密码加密绑定到 Windows 用户账户，不同用户之间无法互相查看密码。
- **启用屏幕锁**。离开电脑时按 `Win + L` 锁屏。
- **使用 Chrome 的个人资料功能**。Chrome 支持创建多个个人资料（类似多个独立的浏览器），每个个人资料的密码互相隔离。
- **定期清除浏览数据**。虽然清除浏览数据不会删除保存的密码（除非你手动勾选），但可以清除其他可能泄露信息的痕迹。关于清除数据的详细方法，参见 [Chrome 清除缓存](/tips/chrome-clear-cache/)。

### Chrome 密码和 Windows 凭据管理器的关系

在 Windows 系统上，Chrome 通过 DPAPI 加密密码，加密密钥存储在 Windows 凭据管理器中。这意味着如果你重装了 Windows 或更换了用户账户，Chrome 中保存的密码将无法解密。如果你需要在重装系统前保留密码，务必提前导出或迁移。

同样的原理也适用于 macOS 上的 Keychain 和 Linux 上的 Keyring。

## Chrome 密码安全和隐私的最佳实践

基于对 Chrome 密码管理器的深入分析，以下是推荐的安全实践：

**1. 开启密码泄露检查并定期使用**

至少每两个月运行一次密码泄露检查。如果 Chrome 提示某些密码已泄露，立即更换。

**2. 为每个重要网站使用独立强密码**

邮箱、银行、社交媒体等高价值账号必须使用独一无二的强密码。Chrome 的密码生成器可以帮助你创建随机密码。虽然 Chrome 的生成器自定义选项不多，但生成 16 位以上的随机字符密码已经足够安全。

**3. 考虑启用两步验证（2FA）**

即使密码泄露，两步验证也能提供额外保护。Chrome 本身不提供 2FA 功能，但 Bitwarden 可以生成和管理 TOTP 验证码。结合使用效果最佳。

**4. 不要长期保留导出的 CSV 文件**

如果确实需要导出密码（比如在完全断网的环境下迁移），完成后立刻用安全删除工具擦除文件。Windows 自带的删除只是移除文件索引，数据仍然可以恢复。

**5. 评估是否需要升级到专业密码管理器**

如果你的密码数量超过 50 个，或者你需要在 Chrome 之外（手机浏览器、Firefox、Safari）使用密码自动填充，建议升级到 Bitwarden 或 1Password。这个投入在安全性上的回报远超成本。

**6. 保持 Chrome 和系统更新**

Chrome 的密码安全依赖于操作系统级别的加密机制。保持 Windows/macOS 更新可以确保加密模块的安全补丁及时生效。同时 Chrome 自身的更新也可能包含密码管理器的安全改进。

## 总结

Chrome 的密码管理器是一个方便的入门级工具，它为你提供了密码保存、自动填充、同步和泄露检查等核心功能。对于密码数量不多、只用 Chrome 浏览器的用户来说，它完全够用。

但你应该清楚地知道它的局限性：不支持跨浏览器使用、没有安全备注和 TOTP、CSV 导出是明文的风险。如果你对密码安全有更高的要求，Bitwarden（免费开源）和 1Password（付费）是值得升级的选择。迁移过程并不复杂，而且不需要经过明文导出的中间步骤。

最重要的是，无论你使用哪个密码管理器，都要养成定期检查密码状态的习惯。打开 Chrome 的密码管理器看看你存了多少密码，运行一次泄露检查，看看有没有需要紧急处理的密码。花 10 分钟做这件事，可能比你想象中重要得多。
