100 个标签页开着不关，不是因为你懒——是真的每个都可能用到。但标签页太多会导致：内存占用爆炸（Chrome 吃 3GB+ 是常态）、找不到之前打开的页面、不小心关了重要的标签页。

59 篇 Chrome 教程里一直缺标签页管理这个高频痛点。这篇给 5 个实用的自动化技巧。

## 一、标签页分组自动整理

### 手动分组

最快的方式——右键一个标签页→向新群组中添加标签页→输入群组名→选颜色。然后把其他同类型标签拖进去。

群组可以折叠——点击群组名折叠整个群组为一个色块，释放标签栏空间。

### AI 自动分组（Chrome 125+）

Chrome 新版本支持基于 AI 的自动分组建议。开启方式：

1. 地址栏输入 `chrome://flags/#tab-group-suggestion`
2. 设为 Enabled
3. 重启 Chrome

启用后，当你打开了多个相关标签页时，Chrome 会自动弹出建议："将这些标签页加入群组？"

### 扩展自动分组

安装 **Tab Groups Extension**，设置规则：
- 域名匹配：所有 `*.github.com` → "开发"群组
- 关键词匹配：标题含"购物" → "购物"群组
- 自定义正则表达式

设置一次规则后，新打开的标签页自动归类。

## 二、标签页休眠释放内存

### 内置休眠（推荐）

地址栏输入 `chrome://settings/performance`：
- 开启"内存节省程序"
- 选择休眠时机：中度（标签页闲置一段时间后）或激进（闲置后立即休眠）

休眠后的标签页会显示一个灰色圆点，点击即唤醒。实测一台 8GB 内存的电脑，开启休眠后从 18 个标签页的 2.8GB 降到 1.4GB。

### 手动休眠指定标签

在不支持自动休眠的场景下，可以用扩展 **The Great Suspender**（手动版）或 **Tab Suspender**：
- 一键休眠当前标签页外的所有标签
- 设置白名单（永不休眠）
- 休眠后标签保持标题可见，不丢失

## 三、批量关闭标签页

### 内置快捷键

| 操作 | 快捷键 |
|------|--------|
| 关闭当前标签 | Ctrl+W |
| 关闭右侧所有标签 | 右键标签→关闭右侧标签页 |
| 关闭其他所有标签 | 右键标签→关闭其他标签页 |
| 重新打开刚关闭的标签 | Ctrl+Shift+T |

### 扩展批量管理

**Tab Manager Plus**——最有用的标签管理扩展之一：
- 按域名分组显示所有标签
- 批量选择→一键关闭
- 搜索标签标题/URL
- 显示每个标签的内存占用

适合每天被上百个标签页淹没的人。

## 四、定时自动刷新

### 扩展方案

**Auto Refresh Plus**：
- 设置刷新间隔（秒/分指定）
- 定时刷新多个标签页
- 支持设置刷新次数限制
- 监控页面变化时自动提醒

适用场景：

| 场景 | 推荐间隔 |
|------|---------|
| 抢票/抢购 | 1-5秒 |
| 股价监控 | 30-60秒 |
| 新闻更新 | 5-10分钟 |
| 后台任务状态 | 30-60秒 |

## 五、一键保存标签页为书签/会话

### 内置书签保存（所有标签）

`Ctrl+Shift+D` —— 保存当前窗口所有标签页为一个书签文件夹。

命名建议：日期+用途，如"2026-07-20-竞品调研"。

### 会话管理器（推荐）

**Session Buddy** 扩展：
- 一键保存当前窗口所有标签为会话
- 支持多个命名会话（工作/学习/娱乐）
- 搜索已保存的会话
- 定期自动备份当前会话
- 崩溃后自动恢复

比内置书签方案好用得多——书签保存了 URL 但丢了浏览状态（滚动位置、表单内容），Session Buddy 保存的是完整会话。

## 六、五个推荐扩展一览

| 扩展 | 用途 | 推荐理由 |
|------|------|---------|
| Tab Groups Extension | 自动分组 | 一次配置长期有效 |
| Tab Suspender | 标签休眠 | 轻量无广告 |
| Tab Manager Plus | 批量管理 | 按域名分组直观 |
| Auto Refresh Plus | 定时刷新 | 简单可靠 |
| Session Buddy | 会话保存 | 标签页管理必装 |

综合方案：Tab Groups Extension（自动分组）+ Tab Suspender（休眠省内存）+ Session Buddy（会话备份）。三个扩展覆盖标签页管理的所有场景，总内存占用不到 100MB。
