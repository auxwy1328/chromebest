网上"Chrome 插件推荐"的文章，动不动就列 30-50 个。问题是你看完之后依然不知道该装哪个——选择太多等于没有选择。

这篇文章只推荐 **10 款插件**，每款我都实际使用超过 3 个月。不堆数量，按使用场景分类，让你知道"我是做XX的，该装哪些"。

## 广告拦截类

### 1. uBlock Origin

**用途：** 拦截网页广告、弹窗、追踪器

**内存占用：** ~45MB（闲置时 20MB）

**为什么推荐它而不是广告拦截管家：** 国内最火的广告拦截插件是 AdGuard 和广告拦截管家。但 uBlock Origin 有三个优势：内存占用更低（比 AdGuard 少 30%）、规则订阅更灵活、完全免费开源。AdGuard 的免费版有部分功能限制，广告拦截管家则会收集用户数据。

**设置建议：** 安装后进入设置 → 过滤器列表 → 勾选"EasyList China"（国内广告规则）+ "EasyPrivacy"（隐私追踪规则）。如果想进一步减少追踪，可以再加 "Fanboy's Annoyance List"。

**替代方案：** 如果你觉得 uBlock Origin 的设置太复杂，可以选 AdGuard（界面更友好但更占内存）。

## 翻译类

![#翻译类](/images/plugins/chrome-essential-extensions/body3.jpg)


### 2. 沉浸式翻译

**用途：** 网页双语翻译、PDF 翻译、视频字幕翻译

**内存占用：** ~80MB（闲置时 40MB）

**核心功能：** 网页翻译、双语对照显示、划词翻译、悬停翻译、PDF 翻译、视频字幕翻译。支持 20+ 种翻译引擎（OpenAI、DeepL、Google、Gemini 等），可以自由切换。

**和同类插件的区别：** Google 翻译只支持整页翻译，没有双语对照功能；DeepL 插件免费版功能受限严重（没有对照翻译和 PDF 翻译）；划词翻译只能划词不能整页翻译。沉浸式翻译是目前功能最全且免费的翻译插件，支持翻译引擎自定义切换是它最大的差异化优势。

**设置建议：** 翻译引擎选 Google（免费稳定），如果需要更好的翻译质量可以配置 DeepL 或 OpenAI API Key。开启"自动翻译"后，访问外文网页会自动翻译。

**详见我们的翻译插件对比文章。**

## 密码管理类

### 3. Bitwarden

**用途：** 密码管理、自动填充、密码生成

**内存占用：** ~25MB

**为什么推荐 Bitwarden 而不是 LastPass 或 1Password：** Bitwarden 是唯一一个完全免费且开源的密码管理器。基础功能（密码存储、自动填充、密码生成器、跨设备同步）全部免费。LastPass 免费版只能在一个设备类型上使用；1Password 需要付费（月费约 3 美元）。

**设置建议：** 注册账号后，导入 Chrome 内置密码管理器中已保存的密码（设置 → 导入数据），然后关闭 Chrome 的密码保存功能，完全交给 Bitwarden 管理。

**替代方案：** 如果你愿意付费，1Password 的界面更美观、家庭共享功能更好。

## 标签页管理类

![#标签页管理类](/images/plugins/chrome-essential-extensions/body2.jpg)


### 4. OneTab

**用途：** 一键合并所有标签页为一个列表，节省内存

**内存占用：** ~15MB

**使用场景：** 当你打开了几十个标签页但暂时不看时，点击 OneTab 图标，所有标签页被关闭并保存为一个列表。你需要时从列表中恢复即可。这是我们内存优化文章中效果最明显的方法。

**和同类插件的区别：** Toby、Workona 等标签页管理插件功能更丰富（标签页分组、工作空间等），但它们会增加内存占用。OneTab 的优势是极简——只有一个功能，内存占用最低。

## 开发工具类

### 5. Wappalyzer

**用途：** 识别网站使用的技术栈（前端框架、后端语言、CDN、分析工具等）

**内存占用：** ~15MB

**使用场景：** 访问任何网站，Wappalyzer 会在地址栏显示该网站使用的技术图标。适合前端开发者和 SEO 从业者了解竞品的技术实现。点击图标可以看到详细的技术清单。

### 6. Vue.js devtools / React Developer Tools

**用途：** 前端框架调试工具

**内存占用：** 各约 20MB

**使用场景：** 如果你做前端开发，对应框架的调试工具是必须安装的。Vue.js devtools 可以查看组件树、Vuex/Pinia 状态、路由信息；React Developer Tools 可以查看组件层级、Props、State。

**注意：** 不做前端开发的人不需要装这些，它们会增加内存占用。如果你同时使用 Vue 和 React，两个 devtools 可以一起安装，它们不会互相冲突。

## 效率工具类

![#效率工具类](/images/plugins/chrome-essential-extensions/body1.jpg)


### 7. Tampermonkey

**用途：** 用户脚本管理器，可以给网页添加自定义功能

**内存占用：** ~25MB

**使用场景：** 通过安装用户脚本（userscript），可以给网页添加各种自定义功能。比如：给百度去掉广告、给 CSDN 自动展开全文、给 GitHub 添加快捷键、给 B站 添加下载按钮等。

**替代方案：** 如果你觉得自己写用户脚本太麻烦，可以搜索 GreasyFork 上已有的脚本——大部分常见需求（去广告、自动展开、快捷键增强等）都有人写好了，直接安装就行。

### 8. Dark Reader

**用途：** 为所有网站启用深色模式

**内存占用：** ~35MB

**使用场景：** 如果你喜欢深色模式但某些网站不支持（比如百度、淘宝），Dark Reader 可以为这些网站强制启用深色模式。效果不错但不是完美的——部分网站的深色模式配色会有些奇怪（图片可能被反转）。

**替代方案：** Windows 10/11 和 macOS 都有系统级深色模式。Chrome 131 也支持跟随系统深色模式。如果你只是想给 Chrome 界面变暗，不需要装 Dark Reader，在 Chrome 设置中选"深色"主题即可。Dark Reader 的价值在于让**不原生支持深色模式的网站**也变成深色。

## 隐私保护类

### 9. Privacy Badger

**用途：** 自动学习和阻止网页追踪器

**内存占用：** ~20MB

**和 uBlock Origin 的区别：** uBlock Origin 基于规则列表拦截（需要手动订阅规则），Privacy Badger 基于机器学习自动判断哪些域在追踪你。两者配合使用效果最好：uBlock Origin 拦截广告，Privacy Badger 拦截隐藏的追踪器。

**设置建议：** 安装后不需要额外设置，Privacy Badger 会自动学习。使用一段时间后（通常 1-2 周），它会自动识别并拦截大部分追踪器。

### 10. ClearURLs

**用途：** 从 URL 中移除追踪参数

**内存占用：** ~5MB

**使用场景：** 很多网站会在 URL 中添加追踪参数（如 `?utm_source=xxx&ref=xxx`），用于追踪你的来源和点击行为。ClearURLs 会在你访问这些链接时自动移除这些参数。内存占用极低（5MB），属于"装了就忘"类型的插件。

## 按场景推荐：你该装哪些？

| 你的场景 | 推荐安装 |
|---------|---------|
| 日常上网 | uBlock Origin + 沉浸式翻译 + Bitwarden |
| 前端开发 | uBlock Origin + Wappalyzer + Tampermonkey + 框架 devtools |
| SEO / 营销 | uBlock Origin + Wappalyzer + Tampermonkey |
| 学生 / 科研 | 沉浸式翻译 + OneTab + Bitwarden |
| 隐私保护重视者 | uBlock Origin + Privacy Badger + ClearURLs + Bitwarden |
| 老电脑（4-8GB内存） | uBlock Origin + Bitwarden（只装这俩，其他都不装） |

**最重要的一个原则：不要装太多插件。** 每个插件都会增加内存占用，拖慢页面加载速度。10 个以上的插件会让 Chrome 明显变慢。建议只安装你每天都会用的 3-5 个插件，其他需要时再临时安装。

**如何判断该不该装一个插件：** 问自己两个问题——"我上周用过这个功能吗？"和"有没有更轻量的替代方案（比如 Chrome 内置功能）？"如果两个问题有一个答案是"否"或"有"，那就不需要装。

## 常见问题

### Chrome 最多能装多少个插件？

Chrome 本身没有限制插件数量，但从性能角度不建议超过 10 个。我们的测试显示：每增加一个插件，Chrome 启动时间增加 0.1-0.3 秒，内存占用增加 15-80MB。10 个插件的 Chrome 比零插件的 Chrome 多占约 300MB 内存。

### 插件会偷看我的数据吗？

有可能。Chrome 插件有访问你浏览数据的权限（取决于插件申请的权限）。安装前查看插件需要的权限：chrome://extensions/ → 点击"详细信息"→ 查看"权限"。如果一个简单的计算器插件要求"读取所有网站数据"，那就是可疑的。尽量选择知名插件（用户量 100 万+）和开源插件。

### 为什么不推荐安装"新标签页美化"类插件？

Chrome 131 的新标签页已经支持自定义壁纸、快捷方式、天气小组件。第三方新标签页插件虽然功能更丰富（待办事项、天气、时钟、灵感壁纸等），但它们会在每次打开新标签页时加载额外资源，增加内存占用（通常 30-40MB）。更重要的是，这类插件有权限访问你的新标签页数据，存在隐私风险。如果确实需要更好的新标签页体验，推荐 Momentum（极简风格，隐私政策透明）。

### 免费插件和付费插件怎么选？

大部分用户需要的核心功能（广告拦截、翻译、密码管理）都有优秀的免费插件。付费插件通常在以下场景下值得考虑：密码管理器（1Password 的界面和体验确实比 Bitwarden 好）、VPN 插件（免费 VPN 速度和安全性都不行）。建议先试免费版，确实觉得不够用再考虑付费。不要因为"贵的一定好"就买付费插件——很多免费插件的质量已经超过付费产品。

### 如何批量备份和恢复插件？

Chrome 插件不能直接导出（只能导出扩展列表，不能导出扩展本身）。备份方法：在 chrome://extensions/ 页面开启"开发者模式"→ 点击"打包扩展程序"手动打包每个插件。恢复时在新电脑上"加载已解压的扩展程序"。但更简单的方法是登录 Google 账号，Chrome 会自动同步大部分插件（不是所有插件都支持同步）。
